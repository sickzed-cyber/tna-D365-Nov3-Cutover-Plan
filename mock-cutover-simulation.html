Full Mock Cutover Simulation

A comprehensive simulation plan for September 8-19, 2025, focusing on coordination between external partners and integration validation before the actual cutover.

Simulation Overview

The full mock cutover simulation will serve as a complete dress rehearsal for the actual cutover planned for October 29 - November 3, 2025. This simulation will validate all technical procedures, coordination mechanisms, and business continuity plans with a core focus on the coordination between external partners.

Simulation Objectives

  • Validate the technical feasibility of the 6-day cutover window
  • Test coordination procedures between all external partners
  • Verify integration points between FINOPS, CRM, Citi Bank, and Vault Design System
  • Identify potential bottlenecks or risks in the cutover process
  • Refine the cutover plan based on simulation results
  • Train the cutover team on their roles and responsibilities
  • Test business continuity procedures during the simulated blackout period

External Partner Coordination

The mock cutover simulation will focus on coordinating activities between the following external partners:

HSO (FINOPS Implementation Partner)

Lead partner responsible for FINOPS implementation and overall cutover coordination.

Key Responsibilities:

  • Lead the overall mock cutover process
  • Coordinate FINOPS data migration activities
  • Configure FINOPS system parameters
  • Set up security roles and user access
  • Implement intercompany processes
  • Manage integration points with other systems

DI (CRM Implementation Partner)

Partner responsible for CRM upgrade and Dual Write connection to FINOPS.

Key Responsibilities:

  • Simulate taking down live CRM system
  • Perform CRM upgrade procedures
  • Prepare CRM for Dual Write connection to FINOPS
  • Configure customer master data synchronization
  • Test quote-to-cash process flows
  • Validate CRM functionality post-upgrade

HSO-Managed Integrations

Citi Bank Integration

Key Responsibilities:

  • Simulate banking system integration procedures
  • Test payment processing workflows
  • Validate bank reconciliation processes
  • Verify transaction security protocols
  • Ensure compliance with banking regulations
  • Test failover and recovery procedures

Vault Design System Integration

Key Responsibilities:

  • Simulate document management workflows
  • Test design document migration
  • Validate version control and approval processes
  • Configure document access controls
  • Test design release freeze and resumption procedures
  • Verify integration with engineering change management

Conclusion

The full mock cutover simulation from September 8-14, 2025 will serve as a critical validation of the cutover plan and coordination procedures between HSO, DI, and integration partners. By simulating the entire cutover process, including FINOPS implementation, CRM upgrade, Dual Write connection, and integrations with Citi Bank and Vault Design System, TNA Solutions will identify and address potential issues before the actual cutover. The lessons learned from this simulation will be incorporated into the final cutover plan, ensuring a smooth transition to D365 on November 3, 2025.
