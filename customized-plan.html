Risk Reduction Measures

Blackout Period Risk Mitigation

To minimize business disruption during the cutover blackout period, the following measures will be implemented:

Vendor Delivery Management

  • Last deliveries accepted: October 24, 2025
  • Vendor communication plan: August 15 - September 15
  • Resume deliveries: November 4, 2025
  • Special handling procedures for critical supplies
  • Emergency procurement process during blackout

Customer Order Fulfillment

  • Early shipment of customer orders where possible
  • Customer communication plan: August 15 - October 15
  • Priority order identification for post-go-live processing
  • Manual order tracking during blackout period
  • Customer service contingency procedures

Design Release Freeze

  • Design freeze begins: October 24, 2025
  • Designs held in Vault during freeze period
  • Resume design releases: November 6, 2025
  • Emergency design release procedure for critical needs
  • Design team training during freeze period

Manual Transaction Processing

  • Templates for urgent transactions during blackout
  • Training on manual procedures: October 12-27
  • Process for loading transactions after November 5
  • Prioritization framework for manual transactions
  • Reconciliation procedures for manual entries

Cutover Execution (October 29 - November 3, 2025)

6-Day Cutover Window

The condensed cutover approach leverages improved data loading capabilities to implement a synchronized global cutover across all entities within a 6-day window.

Phase 1: System Freeze and Initial Data Extraction (October 28, 2025)

  • Implement transaction freeze in legacy systems
  • Complete September month-end close activities
  • Extract initial data from legacy systems
  • Begin using manual transaction templates

Phase 2: Data Migration for All Entities (September 27-29, 2025)

  • Transform and load master data for all entities
  • Load products and supporting data
  • Load BOMs, routings, and inventory data
  • Load open transactions and balances

Phase 3: System Configuration and Validation (September 30 - October 1, 2025)

  • Configure system parameters for all entities
  • Set up security roles and user access
  • Configure workflows and approval processes
  • Validate intercompany processes

Phase 4: CRM Integration and Final Preparation (November 1, 2025)

  • Configure CRM integration for all entities
  • Establish dual-write connections
  • Synchronize customer and product data
  • Conduct final go/no-go assessment

Phase 5: Go-Live (November 3, 2025)

  • Activate D365 for all users globally
  • Implement hypercare support model
  • Monitor system performance and stability
  • Begin post-implementation support

Post-Go-Live Activities

Immediate Post-Go-Live (November 3-6, 2025)

  • Enter all manual transactions collected during blackout
  • Resume normal vendor deliveries (November 4, 2025)
  • Process backlogged customer orders
  • Validate system functionality across all entities
  • Address critical issues through hypercare support

New Product Design Release (November 6, 2025 onwards)

  • Begin releasing new product designs from Vault into D365
  • Resume normal design release processes
  • Validate design-to-manufacturing processes in D365
  • Monitor design release performance
  • Optimize design release workflows
